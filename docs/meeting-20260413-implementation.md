## 変更ファイル一覧

### 新規作成（4ファイル）

#### `columns.html` — コラム一覧ページ
- 4列タイル（モバイル1列 / タブレット2列 / PC4列）
- カテゴリフィルタチップ: すべて / 初めての方向け / 事例・インタビュー / スキル・ノウハウ / 暮らし・働き方 / お知らせ
- 8件のサンプルコラムカード
- 末尾にLINE友だち追加CTA（集客クッション訴求）
- JSでカテゴリフィルタリング実装

#### `register.html` — 会員登録ページ
- 最低限フォーム: 姓/名/メール/電話/生年月日/パスワード
- 利用規約・プライバシーポリシー同意チェック
- 送信時 sessionStorage.loggedIn=1 をセットしてjobs.htmlへ遷移
- 「登録後に職務経歴書・詳細希望条件は応募時に」の案内

#### `apply.html` — 応募フォーム
- 3ステップ（入力→確認→完了）のプログレスインジケータ
- 応募先求人サマリカード
- セクション: 基本情報 / ご経験・ご希望 / 履歴書・職務経歴書（PDF/Word対応、ドラッグ&ドロップ）
- 利用規約同意
- **ログイン済み時の自動補完**:
  - URLパラメータ `?logged_in=1` または sessionStorage.loggedIn で検知
  - ログイン中: 氏名/メール/電話を読み取り専用で自動入力、バナー表示
  - 未ログイン: ログイン導線プロンプトを表示

#### `apply-complete.html` — 応募完了ページ
- チェックマーク + お礼メッセージ
- 今後の流れ（4ステップ）を可視化
- 確認メール送信 + info@チーム通知の文言
- CTA: 他の求人を見る / トップページへ

## 自動補完ロジックの仕組み

```
[login.html / register.html]
  submit → sessionStorage.setItem('loggedIn', '1')
         → jobs.html へリダイレクト
                ↓
[jobs.html → recruit.html → apply.html]
  セッション中はサイト内ナビでloggedIn状態保持
                ↓
[apply.html ロード時]
  URLParams.logged_in === '1' OR sessionStorage.loggedIn === '1'
    ↓ true
  氏名/メール/電話を既定値で自動入力 + 読み取り専用化
  「ログイン中 — 基本情報を自動入力しました」バナー表示
    ↓ false
  「会員の方はログインで自動入力」プロンプト表示（ログインボタン付き）
```

本番実装時は sessionStorage をサーバーサイドのセッション/JWTに置き換え、ダミーの会員データをAPI経由で注入する。
